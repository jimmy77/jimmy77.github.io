So you what exactly is the difference between
"ID's" and "Classes"?

In the world of webpages, html tags help give structure to a website but are limited to how the content can be displayed. It can describe to a browser that a piece of information being shown is either a paragraph or a header, or that a word is bold or italicised.

Basic tags like h2, p, strong and em can give your page structure but can only show you text in black and with a standard system font.

Creating an impactful webpage requires CSS selectors, and the Class and ID play a big role when it comes to visually defining the content it displays.

Class's are NOT unique

  • You can use the same class on multiple elements
  • You can use multiple classes on the same element

Classes are the workhorses of selectors and they do the general heavy-lifting when it comes to styling. In the CSS a class selector is a name that has a full stop before it. You should use classes for items that you know you're going to use a lot.

An example would be if you wanted to give all the paragraphs on your webpage the same styling, you would use classes.

Headlines would also be stylised in the same way. All H1’s will be the same, all H2’s the same, callout boxes.

A good rule of thumb I remember picking up from my design days was if there’s 2 or more elements that the same in terms of their heading levels, then you should apply class selector’s. The benefit of this is that you can have the same HTML element, but present it differently depending on its class or ID. IDs are unique * Each element can have only one id * Each page can have only one element with that id An ID selector appears in the CSS as a selector name with a hash character in front of it. The key difference between the two is that a “Class” can be applied multiple times to can be used several times, while an ID can only be used once.
